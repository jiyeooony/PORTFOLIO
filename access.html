인식의 용의성 (Perceivable) 01

1.1 적절한 대체 텍스트 제공 - 텍스트 아닌 콘텐츠에는 대체 텍스트를 제공해야 한다.

◆ 해당소스

◆ 설명

운용의 용의성 (Operable) 02

2.1 입력장치 접근성 - 2.1.1 키보드 사용 보장 : 모든 기능은 키보드만으로도 사용할 수 있어야 한다.

◆ 해당이미지

◆ 설명

: 키보드 초첨을 받았을 때도 동일하게 기능을 이용할 수 있도록 하였습니다. 마우스나 키보드 어떤 방식으로 접근하더라도 주 메뉴의 세부메뉴까지 확인하고 선택할 수 있습니다.

2.4 쉬운 내비게이션 - 2.4.1 반복 영역 건너뛰기 : 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다.

◆ 해당이미지

◆ 설명

: 본문 바로가기를 제공함으로써 키보드로 웹 페이지를 탐색하는 사용자도 마우스 사용자와 같이 본문으로 바로 이동할 수 있어 콘텐츠를 운용하는데 원활하도록 하였습니다. 반복되는 콘텐츠 영역은 건너뛸 수 있도록 제공함으로써 운용의 용이성을 높였습니다.

이해의 용의성 (Understandable) 03

3.2 예측 가능성 - 3.2.1 사용자 요구에 따른 실행 : 사용자가 의도하지 않은 기능은 실행되지 않아야 한다.

◆ 해당이미지

◆ 설명

팝업 창(레이어팝업 포함)을 쓰지 않고 웹 사이트 초기화면에서 중요한 내용을 사용자들이 잘 보거나 전달 받을 수 있도록 의도하지 않은 실행이 일어나지 않도록 만들었습니다.

3.3 콘텐츠의 논리성 - 3.2.1 콘텐츠의 선형 구조 : 콘텐츠는 논리적인 순서로 제공해야 한다.

◆ 해당이미지

◆ 설명

html에는 로고를 먼저 넣으므로서 논리적인 순서로 선형구조에 맞게 작성하였습니다.

견고성 (Robust) 04

4.1 문법 준수 - 4.1.1 마크업 오류 방지 : 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.

◆ 해당이미지

◆ 설명

: 마크업 검사에서도 통과를 받아 견고성을 지켰습니다.
